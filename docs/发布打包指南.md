# LookinMCP 发布打包指南

本文档面向仓库维护者。目标是产出一个可直接安装的 Lookin.app，并可进一步打成 dmg 分发给外部用户。

## 产物目标

- `Lookin.app`
- `LookinMCP.dmg`

其中 app bundle 必须包含：

```text
Lookin.app
  Contents/
    MacOS/Lookin
    PlugIns/lookin-mcp
```

## 前置条件

- Xcode 15+
- Swift 6 toolchain
- 可用的签名身份
- 如果要提交 notarize，还需要可用的 `notarytool` profile

## 脚本说明

- `scripts/release/build-lookin-mcp-release.sh`
  负责构建 release 版 `lookin-mcp`
- `scripts/release/assemble-lookin-app.sh`
  负责将 helper 注入 app bundle，并执行签名
- `scripts/release/verify-lookin-release.sh`
  负责检查 app 内嵌 helper 和签名状态
- `scripts/release/package-lookin-release.sh`
  负责串联 xcodebuild、helper 注入、签名校验和 dmg 生成

## 本地打包

最短路径：

```bash
scripts/release/package-lookin-release.sh
```

默认输出目录：

```text
build/release/output/
  Lookin.app
  LookinMCP.dmg
```

## 签名与 notarize

本地测试可以使用 ad-hoc 签名。正式分发建议显式传入签名身份：

```bash
scripts/release/package-lookin-release.sh \
  --codesign-identity "Developer ID Application: Your Name"
```

如果要在打包时同时执行 notarize：

```bash
export LOOKIN_NOTARY_PROFILE="your-notary-profile"
scripts/release/package-lookin-release.sh \
  --codesign-identity "Developer ID Application: Your Name" \
  --notary-profile "$LOOKIN_NOTARY_PROFILE"
```

## 发布前检查

至少确认以下几点：

1. `Lookin.app/Contents/PlugIns/lookin-mcp` 存在且可执行
2. `codesign --verify --deep --strict Lookin.app` 通过
3. 安装 app 后，不依赖源码目录也能启动 MCP host
4. `http://127.0.0.1:3846/status` 可访问
5. MCP client 能调用 `lookin.find_nodes`
